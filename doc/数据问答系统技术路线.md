```

  │
  │  POST /api/v1/ai-query  (SSE stream)
  │  Authorization: Bearer <jwt>
  ▼
FastAPI (Python 3.11)
  │
  ├─ [1] Auth middleware → UserContext (role_level, dept_code, user_name)
  ├─ [2] SchemaIndexService → 从 YAML 加载模块索引
```

## 2. 请求生命周期（详细）

### 2.1 前端发起

```typescript
// useAIQuery.ts — submit()
const body = JSON.stringify({ question, history });
const response = await fetch(`${baseUrl}/ai-query`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${token}`,
  },
  body,
});
// Read SSE stream from response.body.getReader()
```

关键点：
- 使用 `fetch()` + `ReadableStream` reader（不用 EventSource，因为需要 POST + Authorization header）
- `chat_history` 从 `localStorage` 读取，每次请求带上最近 6 条
- 用 `AbortController` 支持取消

### 2.2 后端处理流程

```
POST /api/v1/ai-query
  │
  ├─ [1] JWT 鉴权 → UserContext
  ├─ [2] SchemaIndexService(redis=None).load()
  │      └─ 直接从 db_table_index.yaml 解析（无 Redis 缓存）
  ├─ [3] IntentClassifier.classify(question, user_ctx)
  │      ├─ Layer 1: json_schema strict mode (OpenAI structured output)
  │      └─ Layer 2: json_object mode (fallback)
  ├─ [4] ToolRegistry.get_all_tools() → 14 个工具
  ├─ [5] ReactEngine.run(question, intent_result)
  │      ├─ 构建 system prompt（注入 schema_context + user_ctx）
  │      ├─ create_agent(llm, tools, system_prompt)
  │      └─ agent.astream_events() → yield SSE events
  └─ [6] log_qa_session() → 写入 qa_session_log
```

### 2.3 前端 SSE 事件处理

```typescript
// useAIQuery.ts — dispatch()
case "intent_resolved":
  store.setIntentLabel(...);
  store.setStatus("thinking");
  break;

case "step_start":
  store.addStep({ stepNo, thought, action, paramsSummary, status: "running" });
  break;

case "step_done":
  store.updateStep(stepNo, { status: "done", resultSummary });
  break;

case "chart_ready":
  store.addChart(data);  // → currentCharts[]
  break;

case "answer_chunk":
  store.setStatus("streaming");
  store.updateLastAiMessage(msg => ({ ...msg, content: msg.content + data.text_delta }));
  break;

case "done":
  store.setStatus("done");
  store.finalizeRound();  // 复制 currentCharts → message.charts, rounds[].charts
  saveHistory(question, answer);  // 持久化到 localStorage
  break;
```

关键点：
- `answer_chunk` 是增量追加，不是全量替换
- `finalizeRound()` 同时写入 `message.charts` 和 `rounds[].charts`
- 历史记录存 localStorage，下次请求带回

## 3. 后端实现细节

### 3.1 SchemaIndexService

**文件**: `backend/app/services/ai/schema_index.py`

```python
class SchemaIndexService:
    def __init__(self, yaml_path="../doc/db_table_index.yaml", redis=None):
        self._path = Path(yaml_path)
        self._redis = redis  # 可选，实际传 None
        self._index = None

    async def load(self):
        # 直接从 YAML 文件加载（无 Redis）
        with open(self._path, encoding="utf-8") as f:
            self._index = yaml.safe_load(f)
```

四个注入点：
1. `get_module_index_text()` → 意图分类 prompt（~400 tokens）
2. `get_modules_for_intent(intent)` → 代码路由（0 tokens）
3. `get_table_summaries_text(modules)` → ReAct prompt（~800 tokens）
4. `validate_query_tables(sql)` → SQL 黑名单校验（0 tokens）

**坑**：原设计用 Redis 缓存 Schema，但 Redis 不可用时启动报错。改为直接读 YAML，无外部依赖。

### 3.2 IntentClassifier

**文件**: `backend/app/services/ai/intent_classifier.py`

双层 LLM 调用策略：

```python
class IntentClassifier:
    def __init__(self, schema_svc):
        self._client = AsyncOpenAI(
            base_url=settings.LLM_BASE_URL,
            api_key=settings.LLM_API_KEY,
        )
        self._model = settings.LLM_LIGHT_MODEL  # 轻量模型

    async def classify(question, user_ctx) -> IntentResult:
        # Layer 1: json_schema strict mode
        result = await self._call_with_schema(system_prompt, question)
        if result: return result

        # Layer 2: json_object mode (fallback)
        result = await self._call_with_json_object(system_prompt, question)
        if result: return result

        # Layer 3: 触发澄清选项
        raise ClassificationError()
```

**关键设计**：
- `_build_json_schema()` 生成的 JSON Schema 必须描述 `time_range` 内部结构（type + start + end），不能只写 `{"type": "object"}`
- `SlotValues.time_range` 使用 `@field_validator(mode="before")` 容错字符串→对象转换

**坑**：LLM 有时输出 `"time_range": "last_week"`（字符串），Pydantic 期望 `{"type": "last_week"}`（对象）。解决方案：三管齐下：
1. Prompt 告诉 LLM 输出对象格式 `{"type": "today"}`
2. JSON Schema 定义 time_range 内部结构
3. Pydantic field_validator 容错字符串自动转换

### 3.3 ReactEngine

**文件**: `backend/app/services/ai/react_engine.py`

```python
class ReactEngine:
    MAX_STEPS = 8
    MAX_LLM_RETRIES = 2

    def __init__(self, llm_base_url, llm_model, schema_context,
                 user_ctx, tools, chat_history=None, llm_api_key=None):
        self._llm = ChatOpenAI(
            model=llm_model,
            base_url=llm_base_url,
            api_key=llm_api_key or "not-needed",
            temperature=0.2,
            streaming=True,
        )

    async def run(question, intent_result):
        agent = create_agent(
            self._llm,
            tools=self._tools,
            system_prompt=system_prompt,
            middleware=[ToolCallLimitMiddleware(run_limit=8)],
        )

        while retries <= MAX_LLM_RETRIES:
            try:
                async for chunk in agent.astream_events(
                    {"messages": [HumanMessage(content=question)]},
                    version="v2",
                ):
                    if event_type == "on_tool_start":
                        yield SSEEvent(type="step_start", ...)
                    elif event_type == "on_tool_end":
                        chart_data = _extract_chart_data(output)
                        if chart_data:
                            yield SSEEvent(type="chart_ready", ...)
                        yield SSEEvent(type="step_done", ...)
                    elif event_type == "on_chat_model_stream":
                        yield SSEEvent(type="answer_chunk", ...)
                break
            except Exception as e:
                retries += 1
                if retries > MAX_LLM_RETRIES or not _is_retryable(e):
                    yield SSEEvent(type="error", ...)
                    return
                await asyncio.sleep(2 ** retries)

        # 空响应保护
        if not answer_buffer.strip() and not chart_ids:
            yield SSEEvent(type="error", {code: "EMPTY_RESPONSE", ...})
            return

        # 只有 LLM 几乎无输出时才注入 fallback 答案
        if not _extract_final_answer(answer_buffer) and chart_ids \
           and len(answer_buffer.strip()) < 100:
            yield SSEEvent(type="answer_chunk", ...)
```

**坑与解决**：

| 坑 | 症状 | 解决方案 |
|---|------|---------|
| LangChain 工具输出格式不固定 | `isinstance(output, dict)` 有时为 False | `_extract_chart_data()` 处理 ToolMessage / JSON string / raw dict 三种格式 |
| LLM 有时 0 步骤直接回答 | 无工具调用，无图表 | System prompt 约束 #8 强制查询数据库 |
| LLM 间歇性空响应 | `astream_events` 0 事件 | `EMPTY_RESPONSE` 检测，主动发 error 给前端 |
| `Final Answer:` 文本泄漏 | 前端看到原始标记 | 前端 `cleanReactArtifacts()` 截断 |
| Fallback 答案重复注入 | 已有完整回答又加一段 | 阈值为 `answer_buffer < 100 字符` |

### 3.4 _extract_chart_data — 处理多样的 LangChain 输出

```python
def _extract_chart_data(output) -> dict | None:
    # 1. Raw dict with chart_id
    if isinstance(output, dict) and "chart_id" in output:
        return output

    # 2. ToolMessage / AIMessage — has .content attribute
    content = getattr(output, "content", None)
    if content is not None:
        if isinstance(content, dict) and "chart_id" in content:
            return content
        if isinstance(content, str):
            try:
                parsed = json.loads(content)
                if isinstance(parsed, dict) and "chart_id" in parsed:
                    return parsed
            except (json.JSONDecodeError, TypeError):
                pass

    # 3. Plain JSON string
    if isinstance(output, str):
        try:
            parsed = json.loads(output)
            if isinstance(parsed, dict) and "chart_id" in parsed:
                return parsed
        except (json.JSONDecodeError, TypeError):
            pass

    return None
```

### 3.5 System Prompt 关键约束

```
## 约束条件
2. 所有数字必须来自本次工具查询结果——绝不编造数据，也绝不从聊天历史中复用数据
8. **最重要**：每个涉及数据统计的问题，都必须至少调用一次工具查询数据库，
   不得凭上下文直接回答。即使聊天历史中有类似数据，也必须重新查询以获取最新值
```

**坑**：LLM 看到聊天历史中有之前的数据回复，会直接复述而不重新查询。约束 #8 解决了这个问题。


### 4.2 MessageBubble — Markdown 渲染 + 图表处理

**文件**: `frontend/src/components/ai/MessageBubble.tsx`

```tsx
import ReactMarkdown from 'react-markdown';
import remarkGfm from 'remark-gfm';

function renderMarkdownContent(content: string): React.ReactNode {
  const processed = cleanReactArtifacts(replaceChartPlaceholders(content));
  return (
    <div className='markdown-body text-sm'>
      <ReactMarkdown remarkPlugins={[remarkGfm]}>
        {processed}
      </ReactMarkdown>
    </div>
  );
}
```

**三步处理流水线**：

1. **`replaceChartPlaceholders`** — 将 `![标题](chart_xxx)` 替换为引用块：
   ```
   > 📊 **标题**（请在右侧思考面板查看）
   ```
   正则：`/^\s*!\[([^\]]*)\]\(chart_[a-zA-Z][a-zA-Z0-9_]*\)\s*$/gm`
   注意：结尾加 `\n` 终止 blockquote，防止后续文字被吞进引用块。

2. **`cleanReactArtifacts`** — 截断 `Final Answer:` 之后的所有内容：
   ```typescript
   const faIdx = content.search(/(?:^|\n)\s*Final Answer:\s*/im);
   if (faIdx !== -1) content = content.slice(0, faIdx);
   ```
   同时删除独立的 `Thought:` / `Action:` / `Observation:` 行。

3. **ReactMarkdown** — 将 Markdown 渲染为 HTML。`remark-gfm` 插件支持表格、删除线、任务列表等 GFM 扩展。

**CSS**：`frontend/src/index.css` 中的 `.markdown-body` 规则覆盖所有元素（table, th, td, code, pre, blockquote, h1-h6, ul, ol, a 等）。

**坑**：
- 图表渲染曾用自定义 `img` 组件拦截 `chart_*` src，后改为完全移除图表（图表只在右侧 ReasoningPanel 显示）
- Markdown blockquote `>` 会吞掉后续行直到空行，必须在替换模板末尾加 `\n`
- `Final Answer:` 文本可能跨 SSE chunk 到达，所以在完整内容上正则裁切而非逐 chunk 处理

**文件**: `frontend/src/hooks/useAIQuery.ts`

SSE 协议格式：
```

```

解析逻辑：以 `\n\n` 分隔事件，每事件提取 `event:` 和 `data:` 行。最后一段可能不完整，留在 buffer 拼接下一个 chunk。

## 6. 关键技术决策

### 6.1 为什么移除 Redis
- Schema 缓存：YAML 文件很小（~35KB），启动时一次性加载到内存足够
- 进化报告：文件存储（md + JSON）更简单，Admin API 直接读文件
- 减少运维依赖：少一个外部服务，少一个故障点

### 6.2 为什么用 react-markdown 而不是 dangerouslySetInnerHTML
- `react-markdown` 是 React 原生组件，安全（不解析 HTML）
- `remark-gfm` 插件支持 GFM 表格语法
- 项目已有完整的 `.markdown-body` CSS 样式

### 6.3 为什么图表只在右侧面板
- 左侧对话区保持文本流畅性，图表占位符替换为文字引用
- 右侧面板集中展示所有轮次的图表，方便对比
- 思考步骤默认折叠，节省空间

### 6.4 为什么用双层意图分类
- Layer 1（json_schema strict）：大多数模型支持，强制结构化输出
- Layer 2（json_object）：兜底，兼容所有 OpenAI-compatible API
- Pydantic field_validator：最后一层防御，容错字符串→对象转换

### 6.5 为什么手动解析 SSE 而不用 EventSource
- `EventSource` 不支持 POST 请求
- `EventSource` 不支持自定义 Authorization header
- `fetch()` + `ReadableStream` reader 提供完全控制

## 7. 常见问题排查

### 7.1 LLM 返回空响应（0 步骤）
症状：`[ReactEngine] 事件循环结束, 共 0 个事件`
原因：LLM API 间歇性返回空（DashScope 偶发）
前端表现：显示 'AI 未产生回复，请稍后重试'
处理：自动触发 EMPTY_RESPONSE 错误，用户可重试

### 7.2 图表不显示
检查清单：
1. 后端日志是否有 `chart_ready` 事件
2. 前端日志是否有 `← SSE 解析事件: chart_ready`
3. ReasoningPanel 的 `currentCharts` 是否有数据
4. 图表占位符 src 是否匹配 `chart_*` 格式

### 7.3 Final Answer 文字泄漏
症状：页面显示 'Final Answer: …' 原始标记
原因：LLM 在流式输出中写入了 ReAct 格式标记
处理：前端 `cleanReactArtifacts()` 自动截断，无需手动操作

### 7.4 表格不渲染
原因：未引入 `remark-gfm` 插件
解决：`<ReactMarkdown remarkPlugins={[remarkGfm]}>`

### 7.5 图表占位符变成空行
原因：早期的 `stripChartPlaceholders` 直接删除整行
解决：改为 `replaceChartPlaceholders`，替换为文字引用块
