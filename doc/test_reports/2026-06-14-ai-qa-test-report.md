## 二、测试问题与结果

### Q1: 计算机学院有哪些班级？每个班级有多少学生？
- **意图类型**: ORG_OVERVIEW_QUERY (moderate)
- **期望答案**: CLASS01软工2024级1班(6人), class2026软工2026(24人)
- **AI实际回答**: 查询了机构总体统计，但**答非所问** — 返回了"各学院学员人数分布"图表，未列出班级名称和人数
- **判定**: ❌ FAIL
- **问题分析**: AI调用了错误的工具（org_overview而非class相关查询），未理解用户询问的是"班级"

### Q2: Python 编程基础的通过率是多少？
- **意图类型**: EXAM_PASS_RATE_QUERY (simple)
- **期望答案**: 3名学生，通过率100%，平均分86.41
- **AI实际回答**: 找到课程CRS_PYTHON_BASE，报告完成率77.78%、3名学员，但**混淆了"完成率"与"通过率"** — 数据库中3人全部通过(is_passed=1)，通过率应为100%
- **判定**: ⚠️ PARTIAL PASS
- **问题分析**: AI找到了正确的课程和数据，但将completion_rate当作pass_rate呈现

### Q3: 系统中有多少学生和多少老师？
- **意图类型**: ORG_OVERVIEW_QUERY (moderate)
- **期望答案**: 51名学生, 3名教师
- **AI实际回答**: "总学习人数：101人...当前系统工具不支持按角色（学生/老师）进行人员分类统计"
- **判定**: ❌ FAIL
- **问题分析**: user_info表有role_level字段可以区分角色，AI未能使用正确的工具查询。报告101人是混淆了course_grade记录数

### Q4: 最近7天（6月4日到6月10日）的学习活跃度趋势如何？

- **期望答案**: 每天活跃10人，趋势稳定

### Q5: 综合成绩不及格的学生有哪些？列出分数最低的前5名。

## 四、发现的问题归类

### 4.1 严重问题（Blocking）

| 问题 | 影响问题 | 严重程度 |
|------|----------|----------|
| **上下文窗口溢出导致澄清循环** | Q7-Q10 | 🔴 Critical |
| 连续6轮对话后，AI开始对所有后续问题（包括简单问题如"有多少门课程"）回复"抱歉，我没有完全理解"并进入无限澄清循环。这导致会话后半段完全无法使用。 | | |

### 4.2 功能缺失

| 问题 | 影响问题 | 严重程度 |
|------|----------|----------|
| **无法区分用户角色** | Q3 | 🟠 High |
| user_info表有role_level字段(0/1/2/3)，但AI报告"不支持按角色统计"。需要检查工具注册表中是否缺少角色统计工具。 | | |
| **课程/班级查询工具缺失或未正确路由** | Q1, Q8 | 🟠 High |
| AI无法查询特定院系下的班级列表，也无法列出某班级的学生名单。需要检查M1_组织架构和M2_用户权限模块的工具覆盖。 | | |

### 4.3 回答质量问题

| 问题 | 影响问题 | 严重程度 |
|------|----------|----------|
| **关键数据仅显示在图表中，文本回答为空** | Q5 | 🟡 Medium |
| 用户问"列出前5名"时，AI将数据放在饼图中，没有文本列表。违反了用户显式要求的展示格式。 | | |
| **混淆"完成率"和"通过率"** | Q2 | 🟡 Medium |
| 数据库中is_passed=1表示通过，但AI报告completion_rate(77.78%)作为通过率。概念混淆。 | | |
| **回答为空（有思考无结论）** | Q6 | 🟡 Medium |
| 思考步骤显示8步完成，但最终答案文本为空。工具返回了数据但未合成为自然语言。 | | |
| **答非所问** | Q1 | 🟡 Medium |
| 用户问"班级"，AI返回了"学院人数分布"图表。意图理解正确但工具选择错误。 | | |

### 4.4 数据准确性问题

| 问题 | 影响问题 | 严重程度 |
|------|----------|----------|
| **将course_grade记录数(101)作为总用户数** | Q1, Q3 | 🟠 High |
| AI多次报告"总学习人数101人"，实际user_info表只有58人。101是course_grade的记录数。 | | |

## 五、建议（未修复）

以下仅为发现问题记录，不包含修复代码：

1. **上下文管理**: 需要在6+轮对话后压缩或截断历史，防止澄清循环
2. **添加角色统计工具**: 需要新增 `query_user_role_distribution` 工具
3. **添加班级查询工具**: 需要新增 `query_class_students` / `query_dept_classes` 工具
4. **回答合成检查**: 需要在ReAct循环结束后检查是否生成了非空自然语言回答
5. **图表与文字平衡**: 当用户明确要求"列出"时，应优先文字列表而非图表
6. **概念对齐**: prompt中需要明确区分"完成率(completion_rate)"和"通过率(is_passed)"
7. **数据来源校验**: course_grade记录数 ≠ 用户数，需要避免此类混淆
